# Agent Link 接入排查

接入失败通常卡在四类问题：Gateway 没启动、MQTT 鉴权失败、Hub 侧租户 auth 未同步、OpenClaw runtime 执行卡住。排查时先看本机一手状态，再看 Hub 日志。

## 客户端检查

先看安装结果镜像和实例状态：

```bash
cat ~/.openclaw/workspace/<agent>/.agent-link/install-result.json
cat ~/.openclaw/channels/aimoo/<agent>/state.json
```

重点字段：

- `install-result.json.status`
- `install-result.json.stage`
- `install-result.json.summary`
- `state.status`
- `state.reason`
- `state.tenantId`
- `state.topic`

常见判断：

- `state.status=online`：接入已成功，问题可能只是 agent 没把结果回给主人。
- `stage=config_written` 或 `gateway_restart_manual` 长时间不变：优先确认 Gateway 是否启动。
- `state.status=reconnecting` / `mqtt_error`：转查 MQTT 鉴权、网络和服务端 Mosquitto 日志。

确认 Gateway：

```bash
ss -ltnp | grep 18789 || true
ps -ef | grep -i 'openclaw gateway' | grep -v grep || true
tail -n 200 ~/.openclaw/logs/gateway.manual.log 2>/dev/null || true
tail -n 200 /tmp/openclaw/openclaw-$(date +%F).log 2>/dev/null || true
```

查 aimoo-link 运行日志：

```bash
grep -RIn 'aimoo:' /tmp/openclaw ~/.openclaw 2>/dev/null | tail -n 100
```

日志判断：

- 有 `bootstrap resolved` 但没有 `mqtt connected`：MQTT 鉴权或网络问题。
- 有 `mqtt connected` 和 `instance online`：接入成功，继续看消息处理。
- 有 `task ... start` 但迟迟不完成：OpenClaw agent 执行或模型超时。

## 服务端检查

如果客户端未在线，看 Hub 容器日志：

```bash
docker logs a2a-hub-api --since 10m 2>&1 | tail -n 200
docker logs a2a-hub-mosquitto --since 10m 2>&1 | tail -n 200
```

重点关键字：

- `POST /v1/agent-link/self-register`
- `POST /v1/agent-link/presence`
- `not authorised`
- `Reloading config`

判断方式：

- `self-register` 成功但 Mosquitto `not authorised`：租户 auth 文件未同步或 broker 未 reload。
- `presence` 持续 200：Hub 已能看到客户端心跳。
- 完全没有 `self-register`：客户端未走到 bootstrap。

检查 Mosquitto auth 文件：

```bash
cat /data/wwwroot/ai-hub/deploy/mosquitto/passwordfile
```

如果仍是占位内容，先修部署和 auth 同步，再继续 MQTT 排查。
