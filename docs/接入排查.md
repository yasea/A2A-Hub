  先说结论：这类现象通常卡在 4 个点之一。

  - 安装脚本写完配置后，Gateway 没真正重启或没拉起来
  - 自注册成功了，但 MQTT 鉴权失败，hub 看不到在线
  - MQTT 已连上，但 OpenClaw runtime 本身没继续执行，所以聊天窗口停住
  - 安装结果只写到了中间态，install-result.json 没被后台检查器推进到最终态

  现在建议你在 hk 上固定按这个顺序查。

  客户端
  先看结果镜像和状态文件，这两个是第一手证据：

  cat ~/.openclaw/workspace/<agent>/.agent-link/install-result.json
  cat ~/.openclaw/channels/aimoo/<agent>/state.json

  重点看这些字段：

  - install-result.json.stage
  - install-result.json.summary
  - state.status
  - state.reason
  - tenantId
  - topic

  如果 state.status=online，说明接入其实已经成功，问题只是 agent 没把结果回给主人。
  如果是 mqtt_error 或 retry_wait，就转去查服务端 Broker。
  如果结果文件停在 config_written 或 gateway_restart_missing，基本就是 Gateway 没起来。

  然后立刻查 Gateway 是否真的活着：

  ss -ltnp | grep 18789
  ps -ef | grep -i 'openclaw gateway' | grep -v grep
  tail -n 200 ~/.openclaw/logs/gateway.manual.log 2>/dev/null
  tail -n 200 /tmp/openclaw/openclaw-$(date +%F).log 2>/dev/null

  如果没有 18789 监听，安装脚本后半段就没跑通，后面再看 hub 没意义。

  再查插件运行日志，确认卡在 bootstrap、mqtt，还是 task 执行：

  grep -RIn 'aimoo:' /tmp/openclaw ~/.openclaw 2>/dev/null | tail -n 100

  常见判断：

  - 有 bootstrap resolved 但没有 mqtt connected：MQTT 鉴权/连接问题
  - 有 mqtt connected 和 instance online：接入成功，问题转到消息处理
  - 有 task ... start 但迟迟不完成：OpenClaw agent 执行或模型超时问题

  服务端
  如果客户端不是 online，直接看 hub 两个容器日志：

  docker logs a2a-hub-api --since 10m 2>&1 | tail -n 200
  docker logs a2a-hub-mosquitto --since 10m 2>&1 | tail -n 200

  重点看：

  - POST /v1/agent-link/self-register
  - POST /v1/agent-link/presence
  - not authorised
  - Reloading config

  判断方式：

  - self-register 200 但 Mosquitto not authorised：租户 auth 文件没同步或没 reload
  - presence 200 持续出现：客户端其实在线
  - hub 完全没 self-register：客户端压根没走到 bootstrap

  另外一定要看：

  cat /data/wwwroot/ai-hub/deploy/mosquitto/passwordfile

  如果还是占位注释，后续所有 MQTT 排查都不用继续了，先修部署。

  最短闭环
  真遇到“中途没响应”时，我建议就跑这组最小命令：

  客户端 hk：

  cat ~/.openclaw/workspace/<agent>/.agent-link/install-result.json
  cat ~/.openclaw/channels/aimoo/<agent>/state.json
  ss -ltnp | grep 18789 || true
  tail -n 100 ~/.openclaw/logs/gateway.manual.log 2>/dev/null || true
  tail -n 100 /tmp/openclaw/openclaw-$(date +%F).log 2>/dev/null || true

  服务端 hub：

  docker logs a2a-hub-api --since 10m 2>&1 | tail -n 100
  docker logs a2a-hub-mosquitto --since 10m 2>&1 | tail -n 100
  cat /data/wwwroot/ai-hub/deploy/mosquitto/passwordfile

  你把这 6 组结果贴出来，基本可以在一轮内锁定是：

  1. Gateway 没起来
  2. MQTT 鉴权失败
  3. 服务端没同步租户 auth
  4. agent 本地执行卡住

  另外我刚才已经把安装脚本逻辑修成了：无 openclaw-gateway.service 时，会自动退回 openclaw gateway run --force，不再直接停在 gateway_restart_missing。这个修复如果还没同步到 hub，hk 上就仍然会复现你说的这种“中途停
  住但没明确报错”。
  下一步最应该做的是先把最新 run.sh 和 routes_integrations.py 同步到 hub，再复测一次。
