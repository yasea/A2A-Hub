# Agent Link 接入排查

接入失败通常卡在四类问题：Gateway 没启动、MQTT 鉴权失败、Hub 侧租户 auth 未同步、OpenClaw runtime 执行卡住。排查时先看本机一手状态，再看 Hub 日志。

## 客户端检查

先看安装结果镜像和实例状态：

```bash
cat ~/.openclaw/workspace/<agent>/.agent-link/install-result.json
cat ~/.openclaw/channels/aimoo/<agent>/state.json
```

重点字段：

- `install-result.json.status`
- `install-result.json.stage`
- `install-result.json.summary`
- `state.status`
- `state.reason`
- `state.tenantId`
- `state.topic`

常见判断：

- `state.status=online`：接入已成功，问题可能只是 agent 没把结果回给主人。
- `stage=config_written` 或 `gateway_restart_manual` 长时间不变：优先确认 Gateway 是否启动。
- `state.status=reconnecting` / `mqtt_error`：转查 MQTT 鉴权、网络和服务端 Mosquitto 日志。

确认 Gateway：

```bash
ss -ltnp | grep 18789 || true
ps -ef | grep -i 'openclaw gateway' | grep -v grep || true
tail -n 200 ~/.openclaw/logs/gateway.manual.log 2>/dev/null || true
tail -n 200 /tmp/openclaw/openclaw-$(date +%F).log 2>/dev/null || true
```

查 aimoo-link 运行日志：

```bash
grep -RIn 'aimoo:' /tmp/openclaw ~/.openclaw 2>/dev/null | tail -n 100
```

日志判断：

- 有 `bootstrap resolved` 但没有 `mqtt connected`：MQTT 鉴权或网络问题。
- 有 `mqtt connected` 和 `instance online`：接入成功，继续看消息处理。
- 有 `task ... start` 但迟迟不完成：OpenClaw agent 执行或模型超时。

## 服务端检查

如果客户端未在线，看 Hub 容器日志：

```bash
docker logs a2a-hub-api --since 10m 2>&1 | tail -n 200
docker logs a2a-hub-mosquitto --since 10m 2>&1 | tail -n 200
```

重点关键字：

- `POST /v1/agent-link/self-register`
- `POST /v1/agent-link/presence`
- `not authorised`
- `Reloading config`

判断方式：

- `self-register` 成功但 Mosquitto `not authorised`：租户 auth 文件未同步或 broker 未 reload。
- `presence` 持续 200：Hub 已能看到客户端心跳。
- 完全没有 `self-register`：客户端未走到 bootstrap。

检查 Mosquitto auth 文件：

```bash
cat /data/wwwroot/ai-hub/deploy/mosquitto/passwordfile
```

如果仍是占位内容，先修部署和 auth 同步，再继续 MQTT 排查。

## 工具脚本

### reset_agent_link.sh — 统一的 Agent Link 状态清理脚本

清理本地 Agent Link 状态（插件、skill、MQTT 配置、openclaw.json 节点）并可选远程注销 Hub 侧记录。

**路径**：`tests/reset_agent_link.sh`

#### 基本用法

```bash
# 清理单个 agent（默认 main）
bash tests/reset_agent_link.sh

# 清理指定 agent
bash tests/reset_agent_link.sh --agent mia

# 清理所有 agent
bash tests/reset_agent_link.sh --all

# 清理所有 agent 并删除插件和 skill
bash tests/reset_agent_link.sh --all --remove-plugin

# 清理所有 agent 并远程注销 Hub 侧记录
bash tests/reset_agent_link.sh --all --remove-remote

# 使用 auth token 远程注销（当本地配置已丢失时）
bash tests/reset_agent_link.sh --all --remove-remote --auth-token <token>
```

#### 参数说明

| 参数 | 说明 |
|------|------|
| `--agent <id>` | 指定要清理的 agent 短 id（默认 main） |
| `--all` | 清理所有 agent 的 Agent Link 状态 |
| `--remove-plugin` | 同时删除 aimoo-link 插件目录和 skill 目录 |
| `--remove-remote` | 远程注销 Hub 侧 agent 记录 |
| `--auth-token <token>` | 使用指定的 auth token 远程注销（需配合 --remove-remote） |
| `--hub-url <url>` | 指定 Hub URL（默认 https://test.aihub.com） |

#### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OPENCLAW_HOME` | `$HOME/.openclaw` | 指定 OpenClaw 配置目录（多用户部署时使用） |
| `HUB_URL` | `https://test.aihub.com` | 指定 Hub 服务地址 |

#### 跨用户执行

脚本会自动检测目标目录的所有者，如果不是当前用户则通过 `sudo` 执行：

```bash
# 以 root 清理 openclaw 用户的数据
OPENCLAW_HOME=/home/openclaw/.openclaw bash tests/reset_agent_link.sh --all --remove-plugin
```

#### 影响范围

- 更新 openclaw.json 中的 channels.aimoo / plugins.aimoo-link 配置
- 删除 channels/aimoo/<agent> 和 workspace*/.agent-link
- 清理 sessions.json 中 sessionId 以 aimoo: 开头的会话
- 删除 TOOLS.md 中 A2A_HUB_AGENT_LINK_BEGIN/END 标记区段
- 删除插件目录 plugins/aimoo-link（--remove-plugin）
- 删除 skill 目录 skills/aimoo（--remove-plugin）
- 远程注销 Hub 侧 agent 记录（--remove-remote）

#### 执行后需要手动操作

```bash
# 重启 Gateway 使配置生效
systemctl --user restart openclaw-gateway.service
```

### test_agent_install_via_chat.sh — 安装测试

端到端测试 Agent Link 插件安装流程：通过 `openclaw agent` 向目标 agent 发送安装命令，验证安装结果、MQTT 上线、remove 命令、批量安装。

**路径**：`tests/test_agent_install_via_chat.sh`

#### 基本用法

```bash
# 测试单个 agent（默认 mia）
bash tests/test_agent_install_via_chat.sh

# 测试多个 agent
TEST_AGENTS="mia kavip xescort" bash tests/test_agent_install_via_chat.sh

# 全量测试（含 remove 和批量安装）
TEST_REMOVE=true TEST_BATCH=true TEST_AGENTS="mia" bash tests/test_agent_install_via_chat.sh
```

#### 参数说明（环境变量）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `HUB_URL` | `https://test.aihub.com` | Hub 服务地址 |
| `OPENCLAW_BIN` | `~/.local/share/nvm/v24.14.1/bin/openclaw` | openclaw CLI 路径 |
| `TEST_AGENTS` | `mia` | 空格分隔的测试 agent 列表 |
| `TIMEOUT_SEC` | `90` | 单个 agent MQTT 上线等待超时（秒） |
| `FORCE_CLEAN` | `true` | 安装前是否清理旧状态 |
| `TEST_REMOVE` | `false` | 是否在安装成功后测试 `aimoo remove` |
| `TEST_BATCH` | `false` | 是否测试批量安装（不传 AGENT_ID） |

#### 测试流程

1. **检查环境**：Hub 健康、openclaw CLI 可用
2. **清理旧状态**：删除 `install-result.json` 和 `state.json`（`FORCE_CLEAN=true` 时）
3. **逐个 agent 安装**：
   - 通过 `openclaw agent --agent <id> -m "..."` 发送安装命令
   - 检测沙箱环境时自动在宿主机执行补装
   - 轮询 `state.json` 等待 MQTT 上线
   - 打印最终状态摘要
4. **测试 remove**（`TEST_REMOVE=true`）：对第一个成功的 agent 执行 remove 并验证清理结果，之后自动重新安装恢复
5. **测试批量安装**（`TEST_BATCH=true`）：不传 AGENT_ID 执行安装脚本，验证是否进入批量模式

#### 环境准备

```bash
# 确保 .env 中的 SERVICE_AGENT_KEYWORDS 已配置好服务检测关键字
# 服务型 agent（名称包含关键字如 assistant/service/服务 等）会自动注册 service
# SERVICE_AGENT_KEYWORDS=assistant/service/服务
# 安装依赖
# jq — 用于解析 JSON 状态文件
which jq || sudo apt install -y jq
```

#### 典型测试流程

```bash
# 1. 先全量重置
bash tests/reset_agent_link.sh --all --remove-plugin
systemctl --user restart openclaw-gateway.service

# 2. 执行安装测试
TEST_AGENTS="mia kavip xescort" TEST_REMOVE=true TEST_BATCH=true \
  bash tests/test_agent_install_via_chat.sh

# 3. 查看结果 — 脚本会输出汇总报告和每个 agent 的在线状态
```

## aimoo CLI 命令参考

### 基础命令

```bash
# 列出所有 agent 状态
openclaw aimoo list
openclaw aimoo list --pending    # 只显示未安装的
openclaw aimoo list --online     # 只显示在线的

# 配置 agent 到 openclaw.json
openclaw aimoo --agent mia setup --connect-url "https://test.aihub.com/agent-link/connect"
openclaw aimoo --agent mia setup --restart --wait --auto-publish-service

# 发布服务
openclaw aimoo --agent kavip publish-service
openclaw aimoo --agent kavip publish-service --title "自定义标题" --summary "简介"

# 注销 agent
openclaw aimoo --agent mia remove
```

### 服务能力检测

安装脚本会自动检测 SOUL.md 中的服务关键字（标题行 + 你是谁/基本身份 区域），并在检测到服务型角色时自动注册 service。

关键字配置：`.env` 中的 `SERVICE_AGENT_KEYWORDS`（逗号分隔，默认：`助手,assistant,客服,service,服务`）

### 批量安装

```bash
# 一键安装所有未接入的 agent（自动检测）
curl -fsSL "https://test.aihub.com/agent-link/install/openclaw-aimoo-link.sh" | bash

# 安装流程：
# 1. 下载并安装 aimoo-link 插件
# 2. 配置所有未接入的 agent 到 openclaw.json
# 3. 重启 Gateway
# 4. 等待所有 agent 上线
# 5. 自动注册服务（如果 SOUL.md 包含服务关键字）
```
